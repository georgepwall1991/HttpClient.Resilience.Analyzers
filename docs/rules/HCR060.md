# HCR060

Dispose `HttpResponseMessage` when using `ResponseHeadersRead`.

## Bad

```csharp
var response = await client.SendAsync(
    request,
    HttpCompletionOption.ResponseHeadersRead,
    cancellationToken);
```

## Good

```csharp
using var response = await client.SendAsync(
    request,
    HttpCompletionOption.ResponseHeadersRead,
    cancellationToken);
```

Returning a stream or content object from the response does not dispose the response owner:

```csharp
var response = await client.SendAsync(
    request,
    HttpCompletionOption.ResponseHeadersRead,
    cancellationToken);

return await response.Content.ReadAsStreamAsync(cancellationToken);
```

## Suppression

Suppress only when ownership is transferred to the caller or another object that guarantees disposal.

## Current Detection

The implementation reports local response variables initialized from awaited visible `HttpClient` calls such as `SendAsync(...)` or `GetAsync(...)` using `HttpCompletionOption.ResponseHeadersRead` unless the declaration is already a `using var`, a `using (...)` ownership pattern, explicitly disposed, directly returned as the response, passed directly into a returned owner wrapper, or passed into a wrapper local that is returned. Locals that only store the pending `Task<HttpResponseMessage>` are skipped so the code fix only targets actual response owners. Resolved custom `HttpClient` types and lookalike calls are skipped unless Roslyn type information shows that the receiver is `System.Net.Http.HttpClient` or the receiver is visibly declared as `HttpClient` in unresolved source.

## Code Fix

The code fix converts a simple local declaration into a `using var` declaration.
