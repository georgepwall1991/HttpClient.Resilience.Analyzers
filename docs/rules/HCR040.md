# HCR040

Do not stack multiple standard resilience handlers.

## Why

Stacking multiple standard resilience handlers can duplicate retries, timeouts, circuit breakers, and rate limiters. That can amplify traffic during outages and make retry behavior difficult to reason about.

## Bad

```csharp
services.AddHttpClient<GitHubClient>()
    .AddStandardResilienceHandler()
    .AddStandardResilienceHandler();
```

## Better

```csharp
services.AddHttpClient<GitHubClient>()
    .AddStandardResilienceHandler();
```

## Current Detection

The first implementation reports duplicate `AddStandardResilienceHandler()` calls in the same fluent registration chain.

## Code Fix

The code fix removes the duplicate `AddStandardResilienceHandler()` call from the fluent chain.

## Suppression

Prefer removing the duplicate handler. Suppress only when the chain is intentionally composed by a custom abstraction and duplicate runtime handlers cannot actually be produced.
